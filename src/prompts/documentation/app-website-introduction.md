# 应用和网站介绍文案撰写指南

## 角色定位
作为自媒体领域写作专家，擅长于创作公众号、小红书等各大平台的文章内容。

## 应用 (App) 介绍模版

### 基础模版

```
[APP 名称] 是一款(简明描述词，如"高效"、"实用"、"功能丰富"的)应用，支持 (安卓 / iPhone / iPad / 多平台) 使用。

(是否免费，如"它完全免费，无需登录即可使用"、"它完全免费，无广告干扰")。

核心亮点包括 (简要描述1、2个关键功能或价值点，如"语音转写准确、字幕悬浮灵活"等)，适用于(使用场景，如"学习、办公、日常生活"等多个场景)，帮助用户大幅提升(关键词，如"效率"、"体验"、"可达性")。
```

### 完整 App 介绍结构

#### 1. 应用基本信息
```
[应用名称] - [简洁的一句话描述]

应用提供者：[开发公司/团队名称]
适配平台：iOS / Android / Web
下载方式：App Store / 华为应用市场 / 官方网站

价格：免费 / 付费 (价格)
首次下载：需要/无需 登录注册
```

#### 2. 核心卖点（3-5 个关键词）
- 卖点 1 及简要说明
- 卖点 2 及简要说明
- 卖点 3 及简要说明
- 卖点 4 及简要说明

#### 3. 使用场景
```
适合[学生 / 职场人 / 创意工作者 / 旅行者等]，在[学习 / 出行 / 外语交流 / 会议记录等]场景中快速提升[效率 / 体验 / 可达性]。
```

#### 4. 界面设计与交互体验

**风格定位**
- 极简科技感 / 扁平设计 / 拟物怀旧 / 极简黑白风 等

**配色基调**
- 主色调：(颜色) - (含义说明)
- 辅助色调：(颜色) - (含义说明)
- 中性色：(颜色) - (说明)

**交互亮点**
- 无打扰弹窗 / 支持手势操作 / 流畅切换模式 / 多视图布局 等

#### 5. 核心功能一览
```
✓ 功能1：[简要描述]，[补充优势说明]
✓ 功能2：[简要描述]，[补充优势说明]
✓ 功能3：[简要描述]，[补充优势说明]
✓ 更多功能……

所有功能都围绕[核心理念]展开设计。
```

#### 6. 使用体验建议
```
无论你是[身份/角色，如"学生"、"职场人"、"旅行者"]，都能获得流畅自然的使用体验。

尤其适合：
- 场景 1 描述
- 场景 2 描述
- 场景 3 描述
```

### 撰写规范

- **字数**: 300 字左右
- **真实性**: 内容必须真实，不能盲目创作
- **模版遵循**: 按照模版规范整理内容
- **去标记**: 最后输出时去掉 []、()、{} 等标记

### 撰写要点

1. **开头吸引**: 用第一句话抓住读者注意力
2. **核心功能优先**: 突出最重要的 2-3 个功能
3. **应用场景具体化**: 给出真实的使用场景例子
4. **用户导向**: 从用户角度说明应用价值，而非技术细节
5. **简洁有力**: 避免冗长的句子，使用短句和要点列举
6. **数据支撑**: 如有用户数、下载量等数据可加入

## 网站介绍模版

### 基础模版

```
[网站名称不能盲目创作，如果没有就用域名填充] 是一个在线(简洁描述词，如"高效"、"智能"、"功能丰富")的网站，(可补充背景，例如："由[机构/团队名称]开发"、"是[某科研项目]的核心成果"、"专为[领域]设计")。

(是否免费，如"它完全免费，无需登录即可使用"、"它完全免费，操作便捷"、"它完全免费，界面简洁，上手零门槛"、"它完全免费，无广告干扰，纯净体验"、"它完全免费，功能强大，使用流畅")。

🛠️ 核心功能

[选择对应场景，填入详细描述]
```

### 场景段落模版库

#### 场景1：通用功能型网站
```
网站提供 [功能描述] 核心功能，适用于(学习、办公、日常生活等)。
用户可通过 [交互特性] ，显著提升 [效率/便利性/体验] 。
```

#### 场景2：专业/垂直工具型网站
```
平台具备 [技术或功能优势，如"精准分析"、"高容错率"、"实时响应"] ，
尤其适用于(古文解析、图像识别、法律文书生成等)，
为(研究人员、教师、内容创作者等)提供强有力的支持。
```

#### 场景3：创作/互动/展示类网站
```
用户可在平台上 (创作插画、发布作品、生成内容) ，同时 (展示方式或分享机制) ，
让内容以更直观、沉浸式的形式呈现。
```

#### 场景4：AI 助手类网站
```
作为一款 (智能/对话/辅助) 型工具，平台可自动完成 (摘要生成/翻译润色/问题解答) 等任务，
适合快速提升用户的 (知识获取效率/写作质量/操作便捷性) 。
```

#### 场景5：教育学习类网站
```
网站提供丰富的 (学习资源/测评工具/教学互动) ，
适用于 (学生、教师、终身学习者) ，
助力系统性掌握 (知识领域或技能) 。
```

#### 场景6：语言文字处理类网站
```
支持 (拼音标注/文本校对/简繁转换/分词分析) 等语言处理功能，
适合 (文本创作者、语言学研究者) ，
提升语言内容处理效率。

#### 场景7：编程/开发辅助类网站
```
平台可自动 (生成代码/格式化脚本/检测错误) ，
适配主流编程语言，
助力开发者高效完成项目开发与调试。
```

#### 场景8：图像/视频处理类网站
```
用户可在线完成 (图片压缩/抠图/滤镜处理/视频剪辑) 等多种操作，
适用于内容创作与媒体输出需求。
```

#### 场景9：文本生成与改写类网站
```
网站基于先进的文本生成模型，支持 (改写润色/文案创作/提纲扩展) ，
为内容创作者与写作者提供高质量文字支持。
```

#### 场景10：知识图谱/信息检索类网站
```
平台通过 (图谱构建/关键词索引/语义搜索) 技术，
帮助用户快速定位、整理和关联复杂信息，
广泛用于 (学术研究、数据挖掘) 等领域。

#### 场景11：心理/情绪调节类网站
```
用户可进行 (情绪记录/冥想练习/AI陪伴) 等操作，
平台提供 (专业引导/沉浸体验) ，
助力心理放松与情绪管理。

#### 场景12：地图/出行/旅游类网站
```
网站内置 (智能路径规划/景点推荐/多人协同) 等功能，
适用于 (自由行规划/路线优化/结伴出行) 场景。
```

用户可创建 (电子表单/反馈问卷/动态表格) 并实时收集数据，

平台内置 (代办事项/倒计时提醒/自动计算器等) 轻量工具，

```

### 撰写规范

- **字数**: 300 字左右
- **真实性**: 内容必须真实准确，不能盲目创作
- **模版规范**: 按照模版规范整理内容
- **去标记**: 最后输出时去掉所有 []、()、{} 标记
- **换行处理**: 注意 "\n\n" 的输入，保证段落清晰

### 输出示例

```
XXX 是一个在线文件格式转换的网站，由专业的开发团队维护。

它完全免费，无需登录即可使用，界面简洁，上手零门槛。

🛠️ 核心功能

平台支持多种格式文件的上传、预览、批量转换，适用于学术整理、办公文档处理、开发测试等数据兼容需求。用户可快速将 PDF、Word、Excel 等格式灵活转换，显著提升文档处理效率。

平台具备智能格式识别和高速处理能力，尤其适用于批量数据处理场景，为研究人员、办公人员、开发者提供强有力的支持。

更多功能如……，均围绕便捷高效的文件处理展开设计。
```

### 常见错误避免

- ❌ "无需登录和注册"（除非这是真的优势，否则不必强调）
- ❌ 过度使用营销词汇，显得不专业
- ❌ 功能描述不具体，停留在"功能丰富"这类模糊表述
- ❌ 过度承诺不能兑现的功能
- ✅ 真实、具体、简洁、有说服力

## JSON 格式化转换指南

### 输出 JSON 结构

```json
{
  "name": "应用或网站名称",
  "categoryId": "category_001",
  "description": "简洁描述信息",
  "tags": ["tag1", "tag2", "tag3"],
  "info": {
    "website_url": "https://example.com",
    "ico_url": "https://example.com/icon.png"
  }
}
```

### 字段说明

| 字段 | 说明 | 取值规则 |
|------|------|--------|
| name | 应用/网站名称 | 从内容中提取，如无则用 URL 替换 |
| categoryId | 分类 ID | 保持原值不变 |
| description | 描述信息 | URL 链接 + 描述，格式为 `<a href="xxx">xxx</a> \n\n 描述信息` |
| tags | 标签 | 保持默认值 |
| info.website_url | 网站 URL | 从表格或内容中的 URL 地址 |
| info.ico_url | 图标 URL | 若无则为空字符串 |

### 批量表格转换示例

#### 输入表格
```markdown
| 名称 | URL | 描述 |
|------|-----|------|
| Gridzzly | http://gridzzly.com/ | 自制网格纸，7 种形状样式 |
| Printablepaper | http://www.printablepaper.net | 在线下载可打印的各类纸张 |
```

#### 输出 JSON
```json
[
  {
    "name": "Gridzzly",
    "categoryId": "category_001",
    "description": "<a href=\"http://gridzzly.com/\">http://gridzzly.com/</a>\n\n自制网格纸，提供 7 种形状样式的网格纸，包括点状样式、横线样式、方格样式、三角网格、蜂巢网格、五线谱等。",
    "tags": ["工具"],
    "info": {
      "website_url": "http://gridzzly.com/",
      "ico_url": ""
    }
  },
  {
    "name": "Printablepaper",
    "categoryId": "category_001",
    "description": "<a href=\"http://www.printablepaper.net\">http://www.printablepaper.net</a>\n\n在线下载可打印的点状、表格打印纸，提供网格纸、横格纸、金融票据、音乐纸等多种纸张类型。",
    "tags": ["工具"],
    "info": {
      "website_url": "http://www.printablepaper.net",
      "ico_url": ""
    }
  }
]
```

### 转换规则

1. **name 字段**: 优先使用应用/网站名称，若无则用 URL 的域名
2. **categoryId**: 根据实际分类设置（如 category_001、category_002）
3. **description 字段**:
   - 第一行必须是 HTML 超链接格式
   - 链接文本和 href 保持一致
   - 后接 `\n\n` 两个换行符
   - 最后跟上具体的描述内容
4. **tags**: 根据分类自动设置合适的标签
5. **website_url**: 从表格或内容中复制原始 URL
6. **ico_url**: 若没有图标 URL，则为空字符串 ""

### 质量检查清单

- [ ] name 字段不为空
- [ ] description 包含 HTML 超链接
- [ ] website_url 是有效的 URL
- [ ] JSON 格式正确，无语法错误
- [ ] 所有字符串值用双引号
- [ ] 数组和对象结构完整
